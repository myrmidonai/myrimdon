## 实例

每个账号

可以创建不同实例(instance/organization)， 

每个实例可以管理多个项目（Project），

每个项目要指定一个工作空间，可以是云端路径（S3等）或本地路径

## 实例成员 （人+AI）

每个实例可以配置 多个 本地运行时（machine runner/runtime， 每个本地运行时内置独立的多个 agent service。
每个 agent spec 独立的角色定位独立的 system prompt， 独立的 workspace memory， 独立的 mcp skill， 这个是类似于新公司建立团队前先规划成员和职责，
每个 agent spec 用于指定工作流节点的任务， 执行前 根据 agent spec 创建 agent instance， 节点结束后销毁 agent instance ， 类似于临时工！


每个实例也可以邀请多个 human 加入， 每个human 可以单独配置自己的 IM 渠道， 人与Agent的所有交互可以自动同步到所有渠道？human不管打开哪个渠道都可以直接work


## 工作流（世界recile）

- 每个任务至少会涉及两个角色 多个流程
    - 示例1： 
        - Human 说 @ArchAgent 帮我创建一个新项目
        - ArchAgent 说 @ArchAgent 我需要更多信息，请提供项目名称、目标和预算
        - Human 说 项目名称：Myrmidon，目标：开发一个AI协作平台，预算：100万
        - ArchAgent 说 @ArchAgent 已收到信息，正在创建项目...
        - Human 说 @ArchAgent 项目创建成功了吗？
        - ArchAgent 说 @ArchAgent 项目已创建成功，项目ID：12345, 路径 /projects/12345
    - 示例2：
        - human 说 @DevAgent git clone xxx 然后增加一个 GET /xxx/health 接口
        - DevAgent 说 @DevAgent 已收到信息，正在执行...
        - DevAgent 说 @DevAgent 已完成，并提交到分支 /feature/health-check， @Qa 请帮忙测试
        - Qa 说 @Qa 已收到，正在测试...
        - Qa 说 @Qa 测试通过，可以合并
        - Human 可选择人工验收并合并/ human 选择 自动验收合并

- 所有工作流都是按PRD4中定义的执行， 
- 所有过程文档或产物 都统一存入 S3 ？ 如果是本地桌面 就是 每个项目的工作空间

- 可视化编辑，支持拖拽节点、连线、配置参数， 也可以通过AI 生成， 反正最终都是 PRD4中的 json/yml
